# DigitalOcean App Platform Deployment Guide

## 🚀 **Deploy to DigitalOcean App Platform**

This guide will deploy both your frontend and backend to DigitalOcean with PostgreSQL database.

## **Prerequisites**
- DigitalOcean account
- GitHub repository (already set up)
- Domain `hammermath.com` (for custom domain)

## **Step 1: Create App on DigitalOcean**

1. **Go to DigitalOcean Apps**: https://cloud.digitalocean.com/apps
2. **Click "Create App"**
3. **Choose "GitHub" as source**
4. **Select Repository**: `KaiSwain/hammer-portfolio-django`
5. **Choose Branch**: `phase1`
6. **Import from existing config**: Choose `.do/app.yaml`

## **Step 2: Configure Environment Variables**

The following environment variables will be auto-configured, but you may need to add:

### **Backend Environment Variables**
```
DJANGO_SECRET_KEY=<auto-generated-secret>
OPENAI_API_KEY=<your-openai-api-key>
OPENAI_MODEL=gpt-4o-mini
DEBUG=False
```

### **Frontend Environment Variables**
```
NODE_ENV=production
NEXT_PUBLIC_API_URL=<auto-generated-backend-url>
```

## **Step 3: Database Setup**

- **PostgreSQL 15** database will be auto-created
- **Connection string** will be auto-injected as `DATABASE_URL`
- **Migrations** will run automatically on deploy

## **Step 4: Custom Domain Setup**

1. **In DigitalOcean Apps Dashboard**:
   - Go to "Settings" → "Domains"
   - Add `portal.hammermath.com`

2. **In your DNS Provider** (where hammermath.com is hosted):
   ```
   Type: CNAME
   Name: portal
   Value: <your-app-domain>.ondigitalocean.app
   ```

## **Step 5: Deploy**

1. **Click "Create Resources"**
2. **Wait for build** (~5-10 minutes)
3. **Check deployment logs** for any issues
4. **Test the application**

## **Troubleshooting**

### **If Build Fails**:
1. Check the build logs in DigitalOcean dashboard
2. Verify all environment variables are set
3. Check that requirements.txt includes all dependencies

### **If Frontend Can't Connect to Backend**:
1. Verify `NEXT_PUBLIC_API_URL` points to backend
2. Check CORS settings in Django
3. Ensure both services are deployed successfully

### **If PDF Generation Fails**:
1. Check if WeasyPrint installed successfully
2. Verify system dependencies in build logs
3. Test with simple HTML first

2. **Deploy DigitalOcean** backend first
3. **Update frontend** to point to new backend
4. **Test thoroughly** before switching domains

6. **Cleanup** old deployments once confirmed working
